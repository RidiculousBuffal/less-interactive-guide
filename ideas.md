# Less 交互式指南 - 设计理念

## 方案一：技术手册风格（Technical Manual）
<response>
<text>
**设计运动**：Brutalist Technical Documentation

**核心原则**：
- 强调代码可读性，代码块占主导地位
- 强烈的黑白对比，以高亮色块标注关键内容
- 网格化布局，左侧目录导航，右侧内容区域
- 仿终端/IDE 风格的代码展示

**色彩哲学**：深色背景（#0d1117 GitHub 暗色调），代码高亮使用 Dracula 主题色系（紫色、绿色、橙色），强调色使用 #58a6ff（蓝色）。情感意图：专业、精准、可信赖。

**布局范式**：三栏布局 - 左侧固定目录（宽度240px），中间内容区域，右侧代码预览面板。

**标志性元素**：
- 终端风格的代码块（带行号）
- 选择器可视化演示区域（实时渲染）
- 章节分隔线使用虚线

**交互哲学**：点击左侧目录平滑滚动到对应章节，代码块支持一键复制，演示区域支持实时交互。

**动画**：页面加载时代码块逐行出现（打字机效果），章节切换时淡入淡出。

**字体系统**：标题使用 JetBrains Mono（等宽字体），正文使用 Inter，代码使用 Fira Code。
</text>

## 方案二：现代教育平台风格（Modern EdTech）
<response>
<text>
**设计运动**：Contemporary Educational Interface with Glassmorphism

**核心原则**：
- 卡片式布局，每个知识点独立成卡
- 渐变背景营造深度感
- 左侧固定导航栏，右侧内容区域
- 交互式代码演示区域（左代码，右效果）

**色彩哲学**：深蓝紫渐变背景（#1a1a2e → #16213e），卡片使用半透明玻璃效果（backdrop-blur），强调色使用 #e94560（红色）和 #0f3460（深蓝）。情感意图：现代、专业、有深度。

**布局范式**：侧边栏导航 + 主内容区域，主内容区域使用卡片网格，代码演示区使用左右分屏。

**标志性元素**：
- 玻璃态卡片（glassmorphism）
- 代码与效果的左右对比面板
- 章节进度指示器

**交互哲学**：悬停卡片时有上浮效果，代码可实时编辑并预览效果，支持章节间的平滑过渡。

**动画**：卡片进入时有 scale + opacity 动画，代码高亮有渐变效果。

**字体系统**：标题使用 Space Grotesk，正文使用 DM Sans，代码使用 JetBrains Mono。
</text>

## 方案三：技术博客风格（Technical Blog）
<response>
<text>
**设计运动**：Minimalist Technical Blog with Strong Typography

**核心原则**：
- 极简主义，内容为王
- 强烈的排版层次感
- 固定侧边栏目录，内容区域宽松
- 代码块使用高对比度主题

**色彩哲学**：浅色背景（#fafafa），强调色使用 #1d4ed8（蓝色），代码块背景使用 #1e293b（深蓝灰）。情感意图：清晰、专注、易读。

**布局范式**：左侧固定目录（240px），右侧内容区域（max-width 800px），居中对齐。

**标志性元素**：
- 精心设计的代码块（带语法高亮）
- 交互式演示区域（可点击的 DOM 树）
- 章节标题使用大号字体

**交互哲学**：目录高亮当前章节，代码块支持复制，演示区域支持实时交互。

**动画**：滚动时目录高亮平滑过渡，代码块展开时有高度动画。

**字体系统**：标题使用 Sora（粗体），正文使用 Source Serif 4，代码使用 Fira Code。
</text>

## 选定方案：方案二 - 现代教育平台风格（Modern EdTech）

选择理由：
- 深色背景更适合代码展示，减少视觉疲劳
- 左右分屏的代码演示区域直观展示 Less 效果
- 玻璃态卡片增加视觉层次感，同时保持专业感
- Space Grotesk + JetBrains Mono 的字体组合兼顾美观与可读性
